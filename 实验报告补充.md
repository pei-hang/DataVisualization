## 一、实验代码

本次实验使用 Python 调用 Open-Meteo 天气接口获取南昌市气象数据，并结合 `pandas`、`matplotlib`、`seaborn` 完成关系型数据可视化、分类数据可视化和回归分析。完整代码见 [main.py](/Users/peihang/PycharmProjects/DV/main.py)。按照实验要求，这里选取“1.1 动态双变量散点图”任务代码进行展示与说明。

```python
def plot_dynamic_scatter(client: OpenMeteoClient) -> list[Path]:
    payload = client.forecast(
        hourly=["temperature_2m", "relative_humidity_2m"],
        forecast_days=3,
    )
    df = hourly_frame(payload)
    df["日期"] = df["time"].dt.strftime("%m-%d")
    df["时间段"] = pd.cut(
        df["time"].dt.hour,
        bins=[-1, 5, 11, 17, 23],
        labels=["凌晨", "上午", "下午", "夜间"],
    )

    g = sns.relplot(
        data=df,
        x="temperature_2m",
        y="relative_humidity_2m",
        hue="时间段",
        col="日期",
        kind="scatter",
        palette="viridis",
        alpha=0.75,
        s=60,
        height=4,
        aspect=1,
    )
    g.set_axis_labels("气温 (°C)", "相对湿度 (%)")
    g.fig.subplots_adjust(top=0.82)
    g.fig.suptitle(f"{CITY}未来 3 天逐小时温湿度动态散点图")
    return [save_grid(g, "01_dynamic_scatter.png")]
```

上述代码的实现过程如下。首先，程序通过 `client.forecast()` 调用 Open-Meteo 的预报接口，获取南昌市未来 3 天逐小时的气温和相对湿度数据。其次，利用 `hourly_frame()` 将接口返回的 JSON 数据转换为 `DataFrame`，再从时间字段中提取日期和小时，并按照凌晨、上午、下午、夜间四个时间段完成分类。最后，使用 `seaborn.relplot()` 绘制双变量散点图，以气温作为横轴、相对湿度作为纵轴，并利用 `hue="时间段"` 对不同时间段进行颜色编码，`col="日期"` 将三天数据分列展示。这样既能观察温湿度之间的关系，又能直观看出不同时间段和不同日期的变化特点。

## 三、实验心得

通过本次实验，我进一步掌握了如何使用 Python 对真实天气数据进行获取、清洗和可视化分析。与直接使用教材示例数据相比，Open-Meteo 接口返回的是原始 JSON 格式，需要先根据小时级和天级字段转换为 `DataFrame`，再结合时间处理、字段映射和分类分组等操作，才能为后续绘图做好准备。这一过程让我更深刻地认识到，数据预处理在可视化分析中占据非常重要的位置。

在绘图实践中，我对 `seaborn` 的多种统计图形有了更加系统的理解。`relplot`、`PairGrid`、`FacetGrid`、`lmplot`、`boxenplot` 等图形不仅能够展示数据分布，还能从不同角度揭示变量关系。例如，散点图适合观察双变量相关性，分面图适合比较不同类别，回归图适合分析趋势，残差图则有助于判断模型拟合效果。通过将多种图形组合使用，我体会到可视化不仅是“把图画出来”，更重要的是“让图准确表达数据特征”。

此外，本次实验也让我意识到代码健壮性的重要性。在实现过程中，不同图形函数可能存在额外依赖或运行环境差异，因此需要考虑图片保存、异常处理和依赖管理等问题。只有把数据获取、绘图逻辑和输出流程统一封装，整个实验代码才能稳定运行。总体来看，本次实验不仅提高了我的数据可视化能力，也增强了我对真实数据分析流程和工程化实现方式的理解。
